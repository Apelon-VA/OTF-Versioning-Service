    public class IdentifyAllConflict
extends ContradictionManagementStrategy
implements Serializable
    Implements the original ACE conflict resolution strategy. This is also used as the default value.

    Essentially this considers conflict to exist when there is more than one state for any given entity on the user's configured view paths.

    Conflict resolution is not performed, so the result of resolution is the same as the passed parameter.

        Determines if there is a conflict as defined by this resolution strategy for the tuples representing an entity. Will return true if there is more than 1 tuple representing different entity states aside from version and path. Will return false if there is less than 2 tuples or the tuples passed only differ in version and path.

        Description copied from interface: ContradictionManagerBI
        Resolves the supplied parts to a conflict resolved latest state.

        Best case this will resolve to one part, however this will depend upon the data and the resolution strategy in use.

        Note that the input list of parts will not be modified by this method.

        NB This method requires that all the parts are from the same entity! If they are not there is no way for this method to determine that and resolution will take place assuming they are all from the same entity.
