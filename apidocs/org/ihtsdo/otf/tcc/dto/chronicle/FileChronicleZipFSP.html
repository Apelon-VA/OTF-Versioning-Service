    public class FileChronicleZipFSP
extends Object
    Using NIO to access a zip file is a really cool idea, but my tests show that the memory footprint is to large, and it is considerably slower than using the Java7 zip utilities directly. So this implementation is left as an example for future testing. See: http://cr.openjdk.java.net/~sherman/zipfs_src/Demo.java http://docs.oracle.com/javase/7/docs/technotes/guides/io/fsp/zipfilesystemprovider.html
