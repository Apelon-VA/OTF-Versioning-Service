9    * different paths have different data for a given component.

13   * "contradiction resolved view" of the component at that time point.
14   */

24      String getDisplayName();

27       * Method to get a description of this conflict resolution strategy.

31       * 
32       * @return The display name of this conflict resolution strategy
33       */
34      String getDescription();

36      /**
37       * Resolves the supplied versions, which may be from more than one entity,
38       * to a conflict resolved latest state.
39       * <p>
40       * Best case this will resolve to one tuple, however this will depend upon
41       * the data and the resolution strategy in use.
42       * <p>
43       * Note that the input list of tuples will not be modified by this method.
44       * 
45       * @param tuples
46       * @return tuples resolved as per the resolution strategy
47       */
48      <T extends ComponentVersionBI> List<T> resolveVersions(List<T> versions);

50      /**
51       * Resolves the supplied parts to a conflict resolved latest state.
52       * <p>
53       * Best case this will resolve to one part, however this will depend upon
54       * the data and the resolution strategy in use.
55       * <p>
56       * Note that the input list of parts will not be modified by this method.
57       * <p>
58       * <strong> NB This method requires that all the parts are from the same
59       * entity! If they are not there is no way for this method to determine that
60       * and resolution will take place assuming they are all from the same
61       * entity. </strong>
62       * 
63       * @param parts
64       * @return parts resolved as per the resolution strategy
65       */
66      <T extends ComponentVersionBI> List<T> resolveVersions(T part1, T part2);
